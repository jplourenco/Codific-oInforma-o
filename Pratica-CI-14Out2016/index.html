Aula prática de 14 de Outubro de 2016

Objectivos

O objectivo desta aula prática é ilustrar a parte da matéria teórica relacionada com a compressão por dicionário.

Parte 1: LZRW1-A

Ross Williams publicou na Internet o seu algoritmo de compressão LZRW1, que é baseado no LZ77. A distribuição contém os seguintes ficheiros:

  • port.h: Definições de tipos de dados.
  • fast_copy.h: Cópia rápida de dados, pode ser optimizada para um dado CPU.
  • lzrw1-a.c: código da função compress: Esta função tem o seguinte cabeçalho: 
    compress( UWORD action, UBYTE *wrk_mem, UBYTE *src_adr,
   ULONG src_len, UBYTE *dst_adr, ULONG *p_dst_len)
    Esta função faz três coisas diferentes de acordo com o parâmetro ACTION: comprime, descomprime e dá informação sobre os parâmetros do compressor.
    • action: COMPRESS_ACTION_IDENTITY, COMPRESS_ACTION_COMPRESS, COMPRESS_ACTION_DECOMPRESS
    • wrk_mem: endereço de um buffer de trabalho
    • src_adr: endereço do buffer onde estão os bytes a comprimir ou descomprimir
    • src_len: comprimento do byffer src_adr
    • dst_adr: buffer para onde é feita a descompressão/compressão
    • p_dst_len: parâmetro de saída - indica o num. de bytes produzidos pelo processo de compressão ou decompressão
    Para mais detalhes, ver o ficheiro compress.h.
  • compress.h: Definições de constantes, tipos de dados, e cabeçalhos da função compress. Deve ser incluído sempre que a função compress é usada.

Considere os programas compr-incompleto.c e decompr-incompleto.c que utilizam a função compress(). Complete estes programas. Pode usar este Makefile (não se esqueça de mudar o nome dos ficheiros para compr.c e decompr.c respectivamente).

Parte 2: LZW

Esta parte do trabalho tem a ver o método LZW (Lempel-Ziv-Welch) para comprimir ficheiros. Esse método está descrito neste artigo que foi publicado no Dr. Dobb's Journal October, 1989 . Recorde que o algoritmo LZW é baseado no algoritmo LZ78.

O programa é invocado, indicando na linha de comando o ficheiro f a comprimir. O comportamento normal é que o programa produz um ficheiro chamado test.lzw com a compressão de f e um chamado test.out resultante da descompressão de test.lzw .

O código C correspondente está no ficheiro lzw-incompleto.c. Pretende-se que o complete.

Parte 3

O ficheiro calgary.zip contém o teste de compressão chamado "Calgary corpus". Este teste contém ficheiros de vários tipos e é uma espécie de "benchmark" dos programas de compressão.

Comprima cada um dos ficheiros do "Calgary corpus", usando os seguintes programas:

  • lzw
  • lzrw1-a
  • gzip
  • bzip2

Para cada ficheiro e para cada um dos programas de compressão, anote a taxa de compressão conseguida e o tempo de execução do programa compressor.
